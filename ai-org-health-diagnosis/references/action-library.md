# 组织氛围改善动作库

## 1. 使用说明

本文件用于指导 AI 根据组织诊断结果生成改善方案。

AI 生成建议时必须匹配：问题类型、影响范围、责任角色、改善周期、验证指标。

禁止输出"加强沟通""提升管理能力""优化机制"等无法执行的泛化建议。所有建议必须回答：谁负责？做什么？什么时候完成？如何验证有效？

## 2. 动作分类模型

```
组织层动作 → 管理层动作 → 员工体验动作
```

## 3. 基础支持类问题动作库

### Action-001：目标不清晰

**触发条件**：Q01 低，或 Q01 + Q02 同时低。

**可能原因**：战略目标没有有效拆解、团队目标传递不足、工作优先级频繁变化。

**管理者动作**：
- 动作：目标对齐会议
- 执行：每季度目标复盘、每月同步重点变化、明确个人目标与团队目标关系
- 负责人：直属主管
- 周期：30 天启动
- 验证指标：Q01 提升、员工目标理解度提升

**HR 动作**：
- 动作：建立目标管理规范
- 包括：OKR/KPI 制定指导、目标沟通模板
- 周期：季度

### Action-002：资源支持不足

**触发条件**：Q02 低。

**可能原因**：工具不足、流程复杂、协作资源不足。

**管理动作**：
- 动作：工作阻塞清理机制
- 执行：建立"影响交付的问题清单"，每周跟进权限/流程/协作问题
- 负责人：团队负责人

**组织动作**：
- 动作：流程优化专项
- 适用：多个部门同时出现 Q02 低
- 验证：流程周期缩短、员工资源满意度提升

## 4. 价值贡献类问题动作库

### Action-003：员工价值感不足

**触发条件**：Q03 低 + Q04 低。

**管理动作**：
- 动作：优势识别与反馈机制
- 执行：主管每月完成员工优势反馈、项目贡献认可

**HR 动作**：
- 动作：贡献认可体系优化
- 包括：项目贡献奖、专业影响力认可、内部分享激励

**验证指标**：Q03 提升、Q04 提升

### Action-004：认可反馈不足

**触发条件**：Q04 低。

| 时间 | 动作 |
|------|------|
| 30 天 | 建立即时反馈习惯（项目阶段表扬、周/月度认可） |
| 60 天 | 建立团队认可机制（优秀案例分享、团队贡献展示） |
| 90 天 | 验证 Q04 提升 |

## 5. 管理关系类动作库

### Action-005：主管管理能力不足

**触发条件**：Q05 低 + Q06 低 + Q07 低。

**管理者动作**：
- 动作：建立 1v1 管理机制
- 要求：每月至少一次，讨论工作状态、困难支持、成长计划

**HR 动作**：
- 动作：管理者能力提升项目
- 包含：反馈训练、教练式沟通、团队管理课程

**验证指标**：30 天 1v1 覆盖率、90 天 Q05-Q07 提升

## 6. 员工成长发展动作库

### Action-006：发展机会不足

**触发条件**：Q10 低 + Q11 低。

**可能原因**：晋升路径不清晰、学习机会不足、成长反馈不足。

**管理动作**：
- 动作：员工发展计划
- 包括：明确成长方向、制定能力提升计划、定期复盘

**HR 动作**：
- 动作：职业发展体系优化
- 包括：职级标准透明化、专家发展通道、培训资源建设

**特殊规则**：若 3-5 年司龄员工 Q10 明显低，优先判断为核心人才成长瓶颈。

## 7. 组织认同类动作库

### Action-007：组织认同不足

**触发条件**：Q08 低 + Q12 低。

**可能原因**：战略传递不足、员工不了解业务价值。

**组织动作**：
- 动作：战略连接机制
- 包括：业务分享会、战略解读、项目价值传播

**验证指标**：Q08 提升、Q12 提升

## 8. 部门专项改善动作

### Action-008：局部组织风险

**触发条件**：三级部门多个指标低于平均 0.5 以上。

**动作流程**：
1. 部门诊断：分析管理方式、工作压力、协作问题
2. 制定 90 天改善计划
3. 复测（周期：3-6 个月）

## 9. 管理者专项动作库

### Action-009：管理者群体提升

**适用**：管理者群体评分低。

**动作**：管理能力提升计划
- 目标管理 → 解决 Q01
- 反馈沟通 → 解决 Q04/Q05
- 员工发展 → 解决 Q06/Q10

## 10. AI 推荐动作规则

AI 生成改善方案时必须按照：

```
问题 → 影响范围 → 责任角色 → 30/60/90 动作 → 验证指标
```

## 11. 动作有效性验证

每个改善动作必须绑定指标：

| 验证层级 | 指标类型 | 示例 |
|----------|----------|------|
| 短期 | 行为变化 | 沟通次数、反馈覆盖率、机制执行率 |
| 中期 | 问卷变化 | 对应 Q 项提升 |
| 长期 | 组织结果 | 人才稳定性、绩效表现、协作效率 |

## 12. 使用原则

- 不针对员工个人提出建议，只针对组织和群体
- 不因为单个低分生成专项动作
- 优先解决影响范围大、改善价值高的问题
- 所有动作必须有验证方式
- 建议优先选择可在 90 天内执行的动作

---

文件目标：让 AI 从"发现组织问题"升级到"帮助 HR 制定组织改善方案"。最终形成：

```
Diagnosis（发现问题）→ Action Library（匹配动作）→ Improvement Plan（生成改善计划）→ Follow-up（验证效果）
```
