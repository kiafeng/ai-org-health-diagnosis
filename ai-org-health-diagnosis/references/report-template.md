AI 组织氛围诊断报告生成模板（权限组织版）

文件定位 本文件定义 AI 组织氛围诊断 Skill 的报告生成规则。
报告生成不是简单根据用户角色输出固定模板，而是根据：

用户身份 + 用户所属组织节点 + 数据权限范围 → 生成对应层级报告 2. 组织权限模型 2.1 组织树模型 所有分析基于三级组织结构：

公司 ├── 一级事业部 │ ├── 二级部门 │ │ └── 三级部门 │ └── ... └── 一级事业部 └── ... 三级部门为最小分析颗粒度。

2.2 数据可见范围规则 CEO / 管理委员会：

权限范围：公司全量数据 可查看：全公司整体健康度、所有一级事业部对比、一级事业部风险分布、全公司关键问题 不可重点展开：单个员工数据、非必要的三级部门明细 关注：组织战略风险 VP / 一级事业部负责人：

权限范围：仅查看自己负责的一级事业部 + 下属全部二级部门 + 三级部门汇总情况 不可查看：其他一级事业部数据 示例： 公司有 A事业部、B事业部、C事业部 VP-A 可查看 A事业部 → A1二级部门 → A1-1三级部门、A1-2三级部门、A2二级部门 VP-A 不能查看 B事业部、C事业部 关注：事业部经营健康和管理改善 部门经理 / HRBP：

权限范围：仅查看自己所在三级部门 可查看：本团队整体结果、本团队维度表现、本团队员工群体分析 不可查看：其他三级部门、同级其他团队详细数据、个人员工数据 关注：团队管理改善 3. 报告生成逻辑 AI 生成报告前必须执行：

Step 1 — 身份识别 确认用户身份：CEO / 一级事业部负责人 / 二级负责人 / 三级部门负责人 / HRBP

Step 2 — 权限计算 根据组织信息确定允许查看范围。

示例：

输入：角色 VP，组织「办公事业部」 系统自动过滤：保留办公事业部及以下，删除其他事业部 Step 3 — 选择报告模板 根据权限范围选择对应模板。

CEO / 管理委员会报告模板
报告目标：公司整体健康如何？哪些事业部存在风险？管理者能力分布如何？组织层面优先解决什么？

整体结构（四板块叙事，参照 McKinsey OHI / BCG Smart Simplicity：诊断 → 排序 → 行动）：

壹 · 执行摘要 — CEO 1 分钟掌握全局

KPI 总览仪表盘（敬业度率、组织能力、健康指数、参与率，hover tooltip 显示计算逻辑）
战略诊断摘要 3 卡片（组织状态总体判断 + 行业定位 + 最核心挑战）
贰 · 全景诊断 — 宏观到中观，诊断"发生了什么" 2.1 行业基准对标 数据来源：直接从 references/industry-benchmarks.md 读取，无需联网检索。

必须包含 6 个对比维度： 序号 对比维度 基准数据 ① Q12 GrandMean 对比 本公司 vs Gallup P50(3.93) vs P75(4.15) vs 公开企业案例 ② 全球及中国敬业度率 本公司推算 vs 中国(20%) vs 全球(20%) vs 标杆(70%) ③ 行业敬业度中位数 本公司 vs 科技/制造/金融等 6 个行业 P25-P75 范围 ④ 公司规模基准 本公司人数所在规模区间 vs 中位数 ⑤ 驱动因素基准 5 大驱动因素行业基准范围 vs 本公司得分 ⑥ 中国市场专项 Gallup 中国数据 + 美世中国指数 vs 本公司

每个对比维度需输出：本公司数值 vs 基准数值、差距、定位判断。

2.2 六大维度雷达图 敬业度 + 5 大组织能力维度，标注 P50/P75 参考线。

2.3 一级事业部健康地图 展示所有一级事业部健康度排序对比（含 vs Gallup P50 差距标注）。

2.4 文化-绩效矩阵（Rule-014） 核心逻辑：健康度是"过程"，绩效是"结果"。二维交叉判断 BU 健康质量。

呈现：SVG 720×400 散点矩阵图（X=绩效代理，Y=健康指数，红色虚线中位数分割线，四象限淡色背景） 四象限：⭐ 标杆引擎 / 🏖 幸福陷阱 / ⚡ 效率型 / ⚠ 危机区 下方配套四象限文字列表卡片。

叁 · 深层诊断 — 诊断"为什么"，结构性分析 3.1 Gallup Q12 层次瓶颈诊断（Rule-012-A） 呈现方式：纯 HTML 文字分层列表（非 SVG 金字塔），瓶颈层红底红框 + 分数标红， 层间差纯数字标注（差距 ≥ 10 红色加粗）。 右侧配套得分表（层得分 + 层间差 + 状态标签）。 下方计算说明小字（Q 题分组逻辑 + Q12 完整题名对照）。

3.2 人群差异洞察 按司龄、职级、性别、绩效等级交叉分析，输出关键群体画像和主要差异。

3.3 管理者效能排名 MES（Rule-013） MES = (Q03+Q04+Q05+Q06+Q07) / 5 × 100/6，按三级部门聚合排名。 呈现：公司 MES 总览 + TOP 5 排名表 + BOTTOM 5 排名表 + 极差分析。

肆 · 根因与行动 — 排序"先解决什么" + 闭环"怎么改善" 4.1 TOP 5 优势与风险 题目级明细，优势项绿色标注，风险项红色标注。

4.2 根因链深度分析（折叠式卡片，Rule-012-B/C/D） 与 SKILL.md Step 5.5 完全一致的执行流程，从离散 Q 项低分追溯到组织结构性根因。 每条根因链一张卡片：信号 → 传导路径 → 根因判断 → 影响评估 → 分层行动。

4.3 系统性 vs 局部问题分类总览

4.4 30/60/90 分层行动计划 时间线卡片布局（红色=30天止血 / 橙色=60天建设 / 绿色=90天固化）。 每条行动含：具体动作 + Owner + 验证指标。

VP / 一级事业部负责人报告模板 数据范围：仅包含当前负责一级事业部。
报告目标：我的事业部健康如何？哪个二级部门需要关注？应该抓什么管理动作？

壹 · 事业部健康总览

（1）KPI 仪表盘 展示当前健康度与公司平均比较，同时标注 vs Gallup P50 差距。各维度展示行业基准范围（从 references/industry-benchmarks.md 读取）。 KPI 卡片：事业部健康指数 / 员工敬业度 / 组织能力 / vs 公司均值，每张卡片含计算逻辑 tooltip。

（2）六大维度对比（事业部 vs 公司） SVG 雷达图（事业部实线 + 公司虚线）+ 数据表（维度 / 事业部 / 公司 / 差异）。

贰 · 结构化诊断

（3）Gallup Q12 层次诊断（本事业部） 计算 L1~L4 四层得分（L1=Q01+Q02 均值，L2=Q03~Q06 均值，L3=Q07~Q10 均值，L4=Q11+Q12 均值），层间差≥10 分判定为瓶颈层。 呈现形式：统一卡片 + 诊断洞察面板（详见 SKILL.md Step 3.6）。

左侧：4 个层级卡片（BU 得分 + 公司均值 + 偏离箭头 + 状态标签 + 瓶颈标识）+ 层间差距标注
右侧：诊断洞察面板（最大瓶颈 / 最大层间差 / 关键发现 / 行动建议）
（4）20 题逐题对比 表格：题号 / 题目 / 事业部 / 公司 / 差异，事业部得分含条形图。

（5）二级部门健康排名 表格：二级部门 / 人数 / 健康指数（含条形图）/ vs 公司均值 / 内部标准差 / 等级标签。

（6）三级团队健康明细（最小颗粒度） 表格：三级团队 / 所属二级部门 / 人数 / 健康指数 / vs 公司均值 / 等级标签。 说明：三级团队为最小管理颗粒度，是 HRBP 日常陪跑的直接对象。

叁 · 风险定位

（7）三级团队健康度横向条形排序图 将本事业部下属三级团队按健康度升序排列，以横向条形图展示（纯 HTML 布局，详见 SKILL.md Step 3.7）。

每行：团队名 → 灰色轨道 + 彩色条形（绿≥85 / 黄≥75 / 橙≥65 / 红<65）→ 人数 + 最弱维度标注
公司均值红色虚线贯穿所有条形
最弱维度：取该团队六大维度中最低分，<70 分标 ⚠
底部 HTML 图例 + 健康指数计算公式
（8）管理者流失风险分析 仅监控 P7+ 管理者（三级部门负责人及以上），不监控普通员工（详见 SKILL.md Step 3.8）。 风险分级：🔴 高危（健康度<60% 且司龄 1-3 年）/ 🟡 需关注（健康度<70% 或司龄<1 年）/ 🟢 稳定（健康度≥70% 且司龄>3 年）。 呈现形式：管理者卡片列表 + 顶部统计栏 + 说明框。

（9）员工群体洞察 按司龄 / 职级 / 性别三维度切分，McKinsey OHI 风格信号面板 + 横向对比条形图（详见 SKILL.md Step 3.9）。

每维度一个面板：顶部信号框（关键发现）+ 主体横向条形图（按健康度升序，四档颜色 + vs 公司偏离箭头）
信号判定：群体间差距≥10 → "分化显著"；最差群体偏离<-5 → "需重点关注"
肆 · 管理动作

（10）经理效能矩阵（二级部门负责人诊断） 核心分析逻辑：基于"均分偏离 × 内部分化"的二维诊断，帮助 VP 识别下属各二级部门的管理者效能特征。

数据权限：仅展示本事业部下属二级部门，不展示其他一级事业部数据。

数据准备：

筛选本事业部下属所有二级部门（n≥5 即展示，n<5 标注"样本不足"；不再使用 n≥20 的硬门槛，小团队的信息也有参考价值） 对每个二级部门计算： X 轴 = 部门均分 - 公司均分（偏离值），X=0 为公司均值参考线 Y 轴 = 部门内员工个人得分均值的标准差（衡量团队内人与人的体验差异，不是题目间差异） 阈值：X=0（公司均值线），Y=20（标准差固定阈值） 散点大小 = 部门员工人数 呈现形式：

① SVG 四象限散点图：

圆点旁直接标注部门名称，引线交替左右排列（按 x 坐标排序后，奇数位在点左侧、偶数位在点右侧），避免标签重叠 详细数据（偏离值 +X.X、标准差 X.X、人数）仅在鼠标悬停 tooltip 中显示 四个象限半透明底色区分（明星经理=浅绿、偏心眼经理=浅蓝、统一型弱经理=浅橙、危险经理=浅红） 四角固定象限标签 ② 四象限解读卡片（SVG 下方）： 按四个象限各一个卡片区域，列出该象限的部门（彩色圆点 + 部门名 + 人数 + 偏离值 + 标准差 + 一句话解读），自动换行排列。

4 象限解读：

象限位置 X（均分偏离） Y（标准差） 名称 含义 干预策略 左下 > 0 < 20 明星经理 高均分 + 体验一致，管理稳定有效 提炼最佳实践，横向推广 左上 > 0 > 20 偏心眼经理 整体不错但内部分化大 重点关注低分个体，缩小团队内差异 右下 < 0 < 20 统一型弱经理 系统性偏低，体验趋同 从管理机制整体提升：目标/激励/管理方式 右上 < 0 > 20 危险经理 低均分 + 严重分化 VP + HRBP 立即联合诊断，可能存在管理方式不当 报告输出内容：

整体结构（四章节叙事，参照 Gallup Q12 团队诊断逻辑 + SHRM 人才保留框架：总览 → 诊断 → 风险 → 改善工具）：

壹 · 团队健康总览

（1）KPI 仪表盘 展示 4 个核心指标：团队健康指数 / vs 公司均值 / 团队人数 / 事业部均值。 每张 KPI 卡片含 hover tooltip 显示计算逻辑（6 分制 → 百分制转换公式）。

（3）优势与改善机会（Rule-021） Gallup 风格双栏布局：

右侧红底=改善机会：团队得分 < 65 或远低于公司 < -5，最多展示 3 项 每项含题目编号、维度名、团队得分、vs 公司差值、一句话定制解读。 无优势/无机会时显示友好占位提示。
