AI 组织氛围诊断报告生成模板（权限组织版）
1. 文件定位
本文件定义 AI 组织氛围诊断 Skill 的报告生成规则。

报告生成不是简单根据用户角色输出固定模板，而是根据：

用户身份 + 用户所属组织节点 + 数据权限范围 → 生成对应层级报告
2. 组织权限模型
2.1 组织树模型
所有分析基于三级组织结构：

公司
├── 一级事业部
│   ├── 二级部门
│   │   └── 三级部门
│   └── ...
└── 一级事业部
    └── ...
三级部门为最小分析颗粒度。

2.2 数据可见范围规则
CEO / 管理委员会：

权限范围：公司全量数据
可查看：全公司整体健康度、所有一级事业部对比、一级事业部风险分布、全公司关键问题
不可重点展开：单个员工数据、非必要的三级部门明细
关注：组织战略风险
VP / 一级事业部负责人：

权限范围：仅查看自己负责的一级事业部 + 下属全部二级部门 + 三级部门汇总情况
不可查看：其他一级事业部数据
示例：
公司有 A事业部、B事业部、C事业部
VP-A 可查看 A事业部 → A1二级部门 → A1-1三级部门、A1-2三级部门、A2二级部门
VP-A 不能查看 B事业部、C事业部
关注：事业部经营健康和管理改善
部门经理 / HRBP：

权限范围：仅查看自己所在三级部门
可查看：本团队整体结果、本团队维度表现、本团队员工群体分析
不可查看：其他三级部门、同级其他团队详细数据、个人员工数据
关注：团队管理改善
3. 报告生成逻辑
AI 生成报告前必须执行：

Step 1 — 身份识别
确认用户身份：CEO / 一级事业部负责人 / 二级负责人 / 三级部门负责人 / HRBP

Step 2 — 权限计算
根据组织信息确定允许查看范围。

示例：

输入：角色 VP，组织「办公事业部」
系统自动过滤：保留办公事业部及以下，删除其他事业部
Step 3 — 选择报告模板
根据权限范围选择对应模板。

4. CEO / 管理委员会报告模板
报告目标：公司整体健康如何？哪些事业部存在风险？组织层面优先解决什么？

第一部分：公司健康总览
展示：组织健康指数（当前得分、健康等级、vs Gallup P50 差距）、五大维度表现（含 P50/P75 参考线标注）。

第二部分：行业基准对标
数据来源：直接从 references/industry-benchmarks.md 读取，无需联网检索。

必须包含 6 个对比维度：

序号	对比维度	基准数据
①	Q12 GrandMean 对比	本公司 vs Gallup P50(3.93) vs P75(4.15) vs 公开企业案例
②	全球及中国敬业度率	本公司推算 vs 中国(20%) vs 全球(20%) vs 标杆(70%)
③	行业敬业度中位数	本公司 vs 科技/制造/金融等 6 个行业 P25-P75 范围
④	公司规模基准	本公司人数所在规模区间 vs 中位数
⑤	驱动因素基准	5 大驱动因素行业基准范围 vs 本公司得分
⑥	中国市场专项	Gallup 中国数据 + 美世中国指数 vs 本公司
每个对比维度需输出：

本公司数值 vs 基准数值
差距（分差/百分点差）
定位判断（低于 P25 / P25-P50 / P50-P75 / 高于 P75）
第三部分：一级事业部健康地图
展示所有一级事业部的健康度对比（含 vs Gallup P50 差距标注）。

第四部分：组织风险 TOP
输出公司级风险。格式：

字段	内容示例
风险	中坚员工成长感下降
影响	研发体系、3-5年员工
证据	Q10 低于公司平均 12%
第五部分：公司级行动建议
输出 HR/管理层动作。

5. VP / 一级事业部负责人报告模板
数据范围：仅包含当前负责一级事业部。

报告目标：我的事业部健康如何？哪个二级部门需要关注？应该抓什么管理动作？

第一部分：事业部健康总览
展示当前健康度与公司平均比较，同时标注 vs Gallup P50 差距。各维度展示行业基准范围（从 references/industry-benchmarks.md 读取）。

第二部分：二级部门健康排名
展示当前一级事业部内部所有二级部门。

第三部分：经理效能矩阵（二级部门负责人诊断）
核心分析逻辑：基于"均分偏离 × 内部分化"的二维诊断，帮助 VP 识别下属各二级部门的管理者效能特征。

数据权限：仅展示本事业部下属二级部门，不展示其他一级事业部数据。

数据准备：

筛选本事业部下属所有二级部门（n≥5 即展示，n<5 标注"样本不足"；不再使用 n≥20 的硬门槛，小团队的信息也有参考价值）
对每个二级部门计算：
X 轴 = 部门均分 - 公司均分（偏离值），X=0 为公司均值参考线
Y 轴 = 部门内员工个人得分均值的标准差（衡量团队内人与人的体验差异，不是题目间差异）
阈值：X=0（公司均值线），Y=20（标准差固定阈值）
散点大小 = 部门员工人数
呈现形式：

① SVG 四象限散点图：

圆点旁直接标注部门名称，引线交替左右排列（按 x 坐标排序后，奇数位在点左侧、偶数位在点右侧），避免标签重叠
详细数据（偏离值 +X.X、标准差 X.X、人数）仅在鼠标悬停 tooltip 中显示
四个象限半透明底色区分（明星经理=浅绿、偏心眼经理=浅蓝、统一型弱经理=浅橙、危险经理=浅红）
四角固定象限标签
② 四象限解读卡片（SVG 下方）： 按四个象限各一个卡片区域，列出该象限的部门（彩色圆点 + 部门名 + 人数 + 偏离值 + 标准差 + 一句话解读），自动换行排列。

4 象限解读：

象限位置	X（均分偏离）	Y（标准差）	名称	含义	干预策略
左下	> 0	< 20	明星经理	高均分 + 体验一致，管理稳定有效	提炼最佳实践，横向推广
左上	> 0	> 20	偏心眼经理	整体不错但内部分化大	重点关注低分个体，缩小团队内差异
右下	< 0	< 20	统一型弱经理	系统性偏低，体验趋同	从管理机制整体提升：目标/激励/管理方式
右上	< 0	> 20	危险经理	低均分 + 严重分化	VP + HRBP 立即联合诊断，可能存在管理方式不当
报告输出内容：

散点图（含部门名标注 + tooltip）
四象限卡片分组展示
重点关注：危险经理象限（右上）的部门 → 30 天内由 VP + HRBP 联合诊断
差异说明：同事业部内最高偏离与最低偏离的差距
注意事项：

仅展示本事业部二级部门，不可暴露其他事业部数据
不得臆造负责人姓名，仅使用数据中实际存在的字段（如 manager 列）
如无 manager 字段则仅展示部门名称，不做额外标注
Y 轴计算必须使用"人员间标准差"（个人得分均值之间的标准差），不能使用"题目间方差"（不同题目得分的方差）
第四部分：三级部门风险定位
仅展示本事业部下属三级部门。用途：帮助 VP 定位问题团队。

第五部分：改善行动计划
输出事业部级 30/60/90 计划。

6. 部门经理 / HRBP 报告模板
数据范围：仅包含当前三级部门。

报告目标：我的团队哪里有问题？员工为什么这样反馈？我下一步做什么？

第一部分：团队健康画像
展示健康指数、公司/事业部位置，标注 vs Gallup P50 差距。

第二部分：团队优势与风险
输出 TOP 优势和 TOP 风险。

第三部分：问题诊断卡
字段	内容示例
问题	认可反馈不足
数据证据	Q04：55 分，低于事业部 10 分
原因判断	可能：反馈机制不足、管理沟通不足
建议动作	30 天：建立 1v1 反馈机制
第四部分：团队改善计划
输出 30/60/90 行动。

第五部分：团队成员绩效-敬业度九宫格（HRBP 专用）
核心分析逻辑：基于员工个人绩效等级和敬业度得分，将团队成员分为不同类别，帮助 HRBP 和团队负责人识别需要关注的个人。

数据要求：

员工绩效等级（A/B+/B/C 或其他等效等级）
员工敬业度得分（百分制）
无历史数据时，「明星倦怠者」归类不可用
分类参照 Rule-011（详见 references/diagnosis-rules.md）：

8 种核心分类：

标签	绩效	敬业度	管理策略
核心贡献者	高（A）	高（≥80）	保留激励
成长型人才	中高（B+）	高（≥80）	培养发展
高产出风险人才	高（A）	低（<60）	关系修复
明星倦怠者	高（A）	下降趋势	防流失
稳定执行者	中（B）	中（60-79）	提升动力
潜在流失者	中高（B+）	低（<60）	重点访谈
待提升员工	低（C）	高（≥80）	能力培养
风险员工	低（C）	低（<60）	改善或调整
补充分类（覆盖所有绩效×敬业度交叉组合，确保每个员工都有归属）：

标签	绩效	敬业度	策略
高产能手	高（A）	中（60-79）	关注趋势
稳定成长	中高（B+）	中（60-79）	鼓励提升
敬业执行者	中（B）	高（≥80）	发挥优势
需关注者	中（B）	低（<60）	了解原因
需改进	低（C）	中（60-79）	辅导计划
呈现形式：

① 散点图：

SVG 散点图，Y 轴从上到下 = 绩效等级 A → B+ → B → C，X 轴 = 敬业度得分 0-100
每个员工一个圆点，按分类着色（核心贡献者=绿、成长型人才=蓝、高产出风险人才/潜在流失者=橙、风险员工=红，其他=灰）
敬业度阈值线：X=80（高敬业度线，绿色虚线）和 X=60（低敬业度线，红色虚线）
② 人员分布统计表：

核心分类在最前，补充分类在后
每行：分类标签 + 颜色圆点 + 人数 + 占比 + 管理策略简述
③ 重点关注提醒：

高风险群体高亮标注：高产出风险人才、潜在流失者、风险员工、需关注者
每个群体一句话行动建议
权限注意：

仅 HRBP / 团队负责人视角可见
不在 CEO 或 VP 报告中展示
不暴露个人员工姓名，仅按分类聚合展示
7. 报告通用输出结构
所有层级报告必须包含：

数据概览 — 发生了什么
问题诊断 — 为什么
优先级判断 — 先解决什么
改善方案 — 怎么做
8. 权限安全规则
AI 必须遵守：

规则 1：禁止跨权限比较。例如 VP-A 不能看到 VP-B 的具体数据
规则 2：禁止输出个人信息，包括员工姓名、单个人回答
规则 3：小样本隐藏。三级部门少于 20 人不输出详细结果
9. 最终报告选择逻辑
IF 用户 = CEO         → 输出：公司级报告
ELSE IF 用户 = VP     → 输出：负责一级事业部报告
ELSE IF 用户 = 部门负责人/HRBP → 输出：负责三级部门报告
END
