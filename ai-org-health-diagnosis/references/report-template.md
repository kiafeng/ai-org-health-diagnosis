# AI 组织氛围诊断报告生成模板

## 1. 文件定位

本文件定义 AI 组织氛围诊断 Skill 的最终报告输出规范。AI 根据不同阅读对象生成不同版本：

- CEO / 管理委员会版
- VP / 一级负责人版
- 部门经理 / HRBP 版

**原则**：同一套问卷数据，不同管理角色看到不同价值。

## 2. 报告生成原则

### 原则 1：一页看懂
每个报告首屏必须回答：当前组织健康状态？最大风险在哪里？最应该优先解决什么？禁止直接展示大量数据。

### 原则 2：从数据到洞察
报告结构：数据表现 → 问题判断 → 原因分析 → 行动建议。

### 原则 3：结论必须有证据
所有判断必须引用指标得分、对比差异、人群特征、组织范围。

## 3. CEO / 管理委员会报告

**使用对象**：CEO、经营委员会、组织管理委员会。

**核心问题**：公司整体健康吗？哪里存在组织风险？下一阶段关注什么？

### 首页：组织健康战略驾驶舱

**模块 1：核心健康指标卡**

| 指标 | 说明 |
|------|------|
| 组织健康度 | (敬业度指数 + 组织能力指数) / 2 |
| 敬业度指数 | 反映员工状态 |
| 组织能力指数 | 反映组织支撑能力 |
| 核心人才健康度 | 分析高绩效员工群体 |

**模块 2：六维健康雷达**

维度：基础需求、管理支持、团队归属、成长发展、组织能力、AI 转型。展示当前得分、公司基准、优势/风险维度。

**模块 3：组织风险地图**

展示一级事业部健康度。

**模块 4：关键风险 TOP 3**

每个风险包含：风险名称、数据证据、影响群体、建议动作。

## 4. VP / 一级负责人报告

**使用对象**：事业部负责人、一级部门负责人。

**核心问题**：我的组织哪里弱？哪个团队影响整体？我要抓什么？

### 首页：事业部健康成绩单
展示六维评分。

### 模块 2：团队横向比较
按三级部门展示。

### 模块 3：管理效能分析

输出四象限：

| 象限 | 特征 | 建议 |
|------|------|------|
| 明星团队 | 高分低波动 | 复制经验 |
| 稳定风险团队 | 低分低波动 | 主管能力提升 |
| 两极分化团队 | 高波动 | 关注公平性和沟通 |
| 高风险团队 | 低分高风险 | 立即介入 |

### 模块 4：事业部改善计划
输出重点问题和 30/60/90 天行动计划。

## 5. 部门经理 / HRBP 报告

**使用对象**：三级部门负责人、HRBP。

**核心问题**：我的团队问题是什么？为什么？我马上做什么？

### 首页：团队健康成绩单
展示总分和公司排名。

### 团队标签（AI 自动生成）

| 标签 | 特征 |
|------|------|
| 高敬业高组织 | 团队状态良好 |
| 有意愿缺资源 | 员工积极，但组织支持不足 |
| 流程好但人心散 | 组织能力高，但员工投入不足 |
| 高风险团队 | 需要优先干预 |

### 模块 2：Q12 题项诊断
展示每道题得分、公司对比、差距，标记最低 3 题。

### 模块 3：问题诊断卡
每个问题生成：问题名称、数据证据、潜台词、原因判断、建议动作。

### 模块 4：人群风险识别
基于司龄、岗位、绩效、管理身份输出风险群体。

### 模块 5：30/60/90 行动计划

## 6. AI 自动生成洞察规则

每个报告必须自动生成三句话总结：

1. **一句话判断**：组织整体健康状态
2. **最大风险**：当前最需要关注的问题
3. **建议优先动作**：下一阶段重点改善方向

## 7. 数据可视化规范

**状态颜色**：绿（健康）、黄（关注）、红（风险）。

**图表优先级**：指标卡 > 雷达图 > 热力图 > 排名图 > 风险列表。避免复杂统计图。

## 8. 权限规则

| 角色 | 查看范围 | 限制 |
|------|----------|------|
| CEO | 全公司数据 | 不可查看个人回答 |
| VP | 本事业部及下属团队 | — |
| 经理 | 自己团队 | 禁止查看其他团队 |

## 9. 最终输出目标

AI 报告最终必须回答：

1. 组织健康状态如何？
2. 哪些地方值得关注？
3. 问题影响谁？
4. 为什么发生？
5. 应该怎么改善？
6. 多久验证？

最终形成：数据分析 → 组织洞察 → 管理决策 → 改善行动 的完整闭环。
