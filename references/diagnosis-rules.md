组织氛围诊断规则库
1. 使用说明
本规则库用于辅助 AI 根据 Q12 指标组合判断潜在组织问题。规则不是绝对结论，AI 必须结合组织差异、员工画像、数据规模形成最终判断。

2. 单指标组合诊断规则（Rule-001 ~ 006）

以下 6 条规则覆盖 Q12 最常见的问题模式，按 Q 项组合判定，每条规则输出"问题名称 → 可能原因 → 建议动作"。具体改善动作的验证指标见 references/action-library.md。

| 规则 | 问题名称 | 指标组合 | 可能原因 | 关键动作 |
|------|----------|----------|----------|----------|
| 001 | 基础支持缺失 | Q01↓ + Q02↓ | 目标传递不足、资源/流程阻塞 | 明确优先级，定期同步目标；优化资源支持机制 |
| 002 | 价值感不足 | Q03↓ + Q04↓ | 工作与能力不匹配、认可缺失 | 增加优势识别，建立反馈习惯；优化认可机制 |
| 003 | 管理反馈缺失 | Q04↓ + Q05↓ | 管理者反馈缺位 | 重点检查是否集中在某部门或管理层级 |
| 004 | 管理质量风险 | Q05↓ + Q06↓ + Q07↓ | 管理者角色未升级、授权不足 | 建立 1v1 机制，增加团队沟通；管理者培训 |
| 005 | 成长体验不足 | Q10↓ + Q11↓ | 发展通道不畅 | 优化职业路径，增加成长机会（若司龄 3-5 年群体明显↓ → 中坚人才发展瓶颈） |
| 006 | 组织认同不足 | Q08↓ + Q12↓ | 战略传递不足、业务价值连接弱 | 加强目标沟通，增强业务价值连接 |

使用注意：AI 必须结合组织差异、员工画像、数据规模综合判断，不能仅凭单条规则下结论。

6. 人群分析规则
Rule-007：成长阶段断层
司龄分析：若新人评分高但 3-5 年员工评分明显下降 → 判断为成长阶段断层。重点：晋升路径、职业发展。

Rule-008：绩效反馈关联
高低绩效差异：若高绩效员工 Q03/Q04 高而低绩效员工明显低 → 判断为绩效反馈和能力发挥存在关联。

7. 团队成员分类规则（经理人 视角专用）
Rule-011：绩效-敬业度人才九宫格
触发条件：经理人 或三级部门负责人生成团队报告，且数据包含绩效字段。

计算方式：

X 轴：员工敬业度得分（百分制）
Y 轴：员工绩效等级（A / B+ / B / C）
阈值：敬业度高 ≥ 80，敬业度中 60-79，敬业度低 < 60
绩效映射：A = 高，B+ = 中高，B = 中，C = 低
8 种核心分类：

标签	绩效	敬业度	管理策略
核心贡献者	高（A）	高（≥80）	保留激励：薪酬倾斜、关键项目委任、职业晋升通道
成长型人才	中高（B+）	高（≥80）	培养发展：高潜计划、导师制、轮岗机会
高产出风险人才	高（A）	低（<60）	关系修复：1v1 深度沟通、识别不满根源、调整管理方式
明星倦怠者	高（A）	下降趋势	防流失：减轻负荷、确认价值、探讨发展路径（需历史数据对比）
稳定执行者	中（B）	中（60-79）	提升动力：明确成长路径、增加认可反馈、赋予挑战任务
潜在流失者	中高（B+）	低（<60）	重点访谈：了解不满原因、调整预期、提供发展承诺
待提升员工	低（C）	高（≥80）	能力培养：技能培训、绩效辅导、人岗匹配评估
风险员工	低（C）	低（<60）	改善或调整：明确改进期望、设定观察期、评估去留
补充分类（未覆盖的交叉组合，用于团队内所有员工都能归入某个类别）：

标签	绩效	敬业度	管理策略
高产能手	高（A）	中（60-79）	关注趋势：留意敬业度是否有下滑迹象，适度认可
稳定成长	中高（B+）	中（60-79）	鼓励提升：提供成长机会，帮助向核心贡献者转化
敬业执行者	中（B）	高（≥80）	发挥优势：识别潜力和瓶颈，考虑绩效提升路径
需关注者	中（B）	低（<60）	了解原因：是否存在管理问题或个人原因，避免滑入风险区
需改进	低（C）	中（60-79）	辅导计划：针对性能力提升和绩效改进方案
数据要求：

需要员工级别的绩效等级和敬业度得分
无历史数据时，「明星倦怠者」归类不可用，该类员工归入「高产能手」或「高产出风险人才」
输出要求：

散点图（Y=绩效等级，X=敬业度得分），每人一个点，按分类着色
人员分布统计表（每类人数 + 占比 + 管理策略）
重点关注提醒（高风险类别：高产出风险人才、潜在流失者、风险员工、需关注者）
注意：该分析仅在 经理人 / 团队负责人视角呈现，不在 CEO 或 VP 报告中展示（涉及个人员工分类，不适合高层视角）。

8. CEO 根因链合成规则（CEO 视角专用）
CEO 的诊断不能停留在"哪个群体得分低"层面，必须从现象上升到结构解释，从离散问题追溯到根因链。本章定义根因链的合成逻辑、分类框架和输出格式。

核心原则
多个低分指标 + 多个受影响群体 → 是否共享同一根因？
                                ↓
              是 → 合并为一条根因链（减少噪音，聚焦杠杆点）
              否 → 保持独立诊断
禁止将同一根因的不同表象拆成多条独立诊断。例如：

Q04(认可表扬) 低 + 1年以内员工健康度低 + P8中层管理健康度低 → 是同一条根因链的三个侧面，不是三个独立问题
因为根源都是"组织缺乏制度性反馈文化"
Rule-012-A：Gallup Q12 层次瓶颈诊断
原理：Gallup Q12 的四层递进结构（基本需求 → 管理支持 → 团队协作 → 成长发展），下层是上层的基础。下层断裂会导致上层无法传导，投入产出比骤降。

触发：CEO / VP 视角报告生成时，必须执行本规则。CEO 使用公司级数据，VP 使用本事业部数据并增加 vs 公司均值对比和诊断洞察面板。

计算：

按 Q12 四层模型聚合各层得分（百分制）：
L1 基本需求 = (Q01 + Q02) / 2
L2 管理支持 = (Q03 + Q04 + Q05 + Q06) / 4
L3 团队协作 = (Q07 + Q08 + Q09 + Q10) / 4
L4 成长发展 = (Q11 + Q12) / 2
计算相邻层之间的分差。分差 ≥ 10 分 → 瓶颈层。
瓶颈层判定：
若 L1 → L2 断层 ≥ 10 分：管理支持是瓶颈。基础牢固但管理杠杆失效，后续投入被 L2 抵消。
若 L2 → L3 断层 ≥ 10 分：团队协作是瓶颈。管理者个体尚可，但跨团队协作和归属感断裂。
若 L3 → L4 断层 ≥ 10 分：成长发展是瓶颈。团队运转正常但长期发展通道不畅。
输出要求：

瓶颈层判定结论（哪一层断裂、断裂程度）
瓶颈层对上层的影响评估（"因为 L2 断裂，在 L3/L4 上的投入传导效率降低"）
呈现方式：纯 HTML 文字分层列表（非 SVG 金字塔）— 四层从上到下排列，瓶颈层红底红框 + 分数标红，层间差纯数字标注（差距 ≥ 10 红色加粗）
右侧配套得分表（层得分 + 层间差 + 状态标签）+ 下方计算说明（Q 题分组逻辑 + Q12 完整题名对照）
VP 视角增强：每张层级卡片增加 BU 得分 + 公司均值 + 偏离箭头（↑/↓）+ 状态标签 + 瓶颈 ⚠ 标识；右侧改为诊断洞察面板（最大瓶颈 / 最大层间差 / 关键发现 / 行动建议），不再使用重复数据表格。
参考模型：Gallup Q12 Meta-Analysis (Harter et al., 2002-2020)，四层递进假设在超过 200 万样本中验证。

Rule-012-B：跨维度关联合成
原理：Rule-001 到 Rule-006 定义了 6 种单维度诊断模式，但它们在实际数据中经常同时触发。跨维度合成将这些离散触发合并为具有结构性解释力的根因链。

合成规则（按优先级排列）：

同时触发的规则组合	合成根因链	结构性解释
Rule-002(价值感不足) + Rule-005(成长体验不足)	反馈闭环断裂	员工不被看见 → 不知道如何成长 → 同时损害价值感和成长体验。不是两个问题，是一个循环断裂。
Rule-003(管理反馈缺失) + Rule-004(管理质量风险)	管理杠杆断在 L2	管理者在日常关怀(Q5)及格，但在发展性管理(Q6/Q7)上系统性缺位。不是"不会管"，是"管理角色未升级"。
Rule-001(基础支持缺失) + Rule-004(管理质量风险)	资源与管理双重缺失	工具不够 + 管理不到位 → 员工既缺硬件又缺软件，优先重度干预。
Rule-005(成长体验不足) + Rule-006(组织认同不足)	内外失衡	组织对外（业务/技术）投入清晰，但对内（人/发展）投入模糊。Denison 模型中适应性 vs 使命的失衡。
Rule-002(价值感不足) + Rule-006(组织认同不足)	组织信任赤字	Great Place to Work 的"尊重"维度系统性薄弱，员工既感受不到个人价值也感受不到组织意义。
使用：在 CEO 报告中，先执行 Rule-001~006 触发判定，再将同时触发的规则按上表合成根因链。每条根因链引用其关联的单个规则作为证据来源。

Rule-012-C：系统性问题 vs 局部问题分类（Burke-Litwin 区分）
原理：Burke-Litwin 模型将组织问题分为两类，CEO 必须区分对待：

转型因子问题（Transformational）：涉及战略、文化、领导力。影响全公司或跨 BU。改变周期长（6-12 个月），需要 CEO 层面决策。
交易因子问题（Transactional）：涉及制度、流程、管理实践。影响特定部门或层级。改变周期短（1-3 个月），可在 BU/HR 层面执行。
分类规则：

特征	系统性（Transformational）	局部性（Transactional）
影响面	全公司或跨多个 BU 低分	单个三级部门或单个 BU 低分
维度模式	某维度全公司得分 < 70 且 ≥ 3 个 BU 同时偏低	某部门单一维度显著偏离（diff < -10）
Q 项模式	某 Q 项全公司排名 BOTTOM 3 且标准差低（全员共识性低分）	某 Q 项仅在该部门异常低而其他部门正常
人群模式	多个人群组（司龄+职级+绩效）同时在该维度低分	仅某一人群组在该部门偏低
对应规则	触发 ≥ 2 条 Rule-001~006 且影响面跨 BU	触发 Rule-009（局部组织风险）
输出要求：

每条根因链标注性质：「系统性」或「局部性」
系统性问题的建议责任人为 CEO/HRVP，动作周期为 60/90/180 天
局部性问题的建议责任人为 BU VP + 经理人，动作周期为 30/60/90 天
Rule-012-D：根因链输出格式
每条根因链在 CEO 报告中必须包含：

根因链名称
├─ 信号（触发了哪些规则、哪些 Q 项低分、哪些群体受影响）
├─ 传导路径（A → B → C，从最底层信号到最上层影响）
├─ 根因判断（结构性解释，不是现象复述）
├─ 性质分类（系统性 / 局部性）
├─ 影响评估（对业务/人才/组织的战略风险）
└─ 分层行动建议
   ├─ 30天：可立即启动的动作 + Owner
   ├─ 60天：中期机制建设 + Owner
   └─ 90天：制度固化 + 验证指标
禁止：根因判断写成现象复述。例如"Q04 低是因为认可不够"是错误的——这只是在用不同措辞重复同一个现象。正确的根因判断是"组织缺乏制度性反馈文化，管理者未被要求也不被评估反馈行为"。

12. 管理者效能排名规则（CEO 视角）

Rule-013：管理者效能排名 (Manager Effectiveness Score, MES)
原理：Gallup 元分析结论——70% 的员工敬业度差异由直接管理者决定。管理者效能排名帮助 CEO 识别组织中的"管理放大器"（正向提升员工体验）和"管理缩小器"（拉低员工体验）。

触发：CEO 视角报告生成时，必须执行本规则。

计算：
- MES = (Q03 发挥优势 + Q04 认可表扬 + Q05 主管关心 + Q06 鼓励发展 + Q07 目标明确) / 5 × 100/6
- 按三级部门聚合（n ≥ 3 即纳入排名，n < 3 标注"样本不足"）
- 管理偏差 = 部门 MES − 公司 MES

输出：
- 公司 MES 总览（数值 + 描述）
- TOP 5 排名表（部门名 + 所属事业部 + MES 得分 + 管理偏差 + 标签"正向管理放大器"）
- BOTTOM 5 排名表（部门名 + 所属事业部 + MES 得分 + 管理偏差 + 标签"需重点关注"）
- 极差分析（TOP − BOTTOM → 管理质量分布的离散度判断）

13. 文化-绩效矩阵规则（CEO 视角）

Rule-014：文化-绩效矩阵 (Culture-Performance Matrix)
原理：组织健康度是"过程指标"，业务绩效是"结果指标"。单一维度无法判断 BU 的健康质量。二维交叉后，CEO 可以区分：健康又高绩效的 BU（可复制）、健康但低绩效的 BU（需激活）、不健康但高绩效的 BU（需修复）、不健康又低绩效的 BU（需干预）。

触发：CEO 视角报告生成时，必须在 BU 健康地图之后执行本规则。

计算：
- X 轴（组织绩效代理）= (Q15 战略对齐 + Q16 敏捷响应 + Q17 跨部门协作 + Q18 流程效率) / 4
- Y 轴（双支柱健康指数）= 员工敬业度(Q1-14 均分×100/6) × 50% + 组织能力(Q15-20 均分×100/6) × 50%（与报告首屏双支柱健康指数一致；注意区别于「敬业度率(Gallup 三分类)」，二者不是同一指标）
- 分割线 = 各维度中位数（X 中位数、Y 中位数）

四象限分类：
| 象限 | 健康度 | 绩效 | 名称 | 管理策略 |
|------|--------|------|------|----------|
| 右上 | ≥ 中位 | ≥ 中位 | ⭐ 标杆引擎 | 提炼最佳实践，横向推广 |
| 左上 | ≥ 中位 | < 中位 | 🏖 幸福陷阱 | 氛围好但不出活，需提升结果导向 |
| 右下 | < 中位 | ≥ 中位 | ⚡ 效率型 | 短期出活但人耗竭，关注可持续性 |
| 左下 | < 中位 | < 中位 | ⚠ 危机区 | 系统性问题，需 CEO 层面干预 |

输出：
- SVG 720×400 二维散点矩阵图：X 轴（绩效代理，刻度 0-100）+ Y 轴（双支柱健康指数，刻度 0-100），红色虚线为中位数分割线，四象限淡色背景区分，右上角/左下角标注象限名称
- 每个 BU 以彩色圆点表示（按象限着色），旁边标注事业部名称
- 下方四象限文字列表（部门名 + 健康度 + 绩效 + 管理提示），一行一个象限组

Rule-015：管理者流失风险分级（VP 视角专用）
原理：管理者（P7+，三级部门负责人及以上）流失的连锁损失远大于普通员工——管理真空 → 下属无方向 → 团队敬业度快速下滑 → 招聘替代成本 3-5 倍 → 下属跟随离职。VP 视角仅监控管理者，不监控普通员工。

触发：VP 视角报告生成时，在三级团队健康明细之后执行本规则。

风险分级（基于健康度 + 司龄双维度）：

| 等级 | 条件 | 含义 | 干预策略 |
|------|------|------|----------|
| 🔴 高危 | 健康度 < 60% 且 司龄 1-3 年 | 已适应但不满，外部机会敏感期 | VP 一对一访谈，了解诉求，制定保留方案 |
| 🟡 需关注 | 健康度 < 70% 或 司龄 < 1 年 | 磨合期不稳定或健康度偏低 | 经理人 每月跟进，关注工作状态 |
| 🟢 稳定 | 健康度 ≥ 70% 且 司龄 > 3 年 | 稳定期，投入产出比高 | 维持现状，发挥传帮带作用 |

输出：
- 顶部统计栏：总人数 + 各等级人数及占比
- 管理者卡片列表：姓名 / 职级 / 所属团队 / 健康度 / 司龄 / 风险等级 + 风险因素说明
- 底部说明框：解释"为何只监控 P7+ 管理者"的原因

参考模型：SHRM Talent Retention Framework + Gallup Q12 Meta-Analysis（管理者敬业度与团队敬业度的传导效应 r=0.6+）。

Rule-016：员工群体分化信号判定（VP 视角专用）
原理：参照 McKinsey OHI Signal-First 方法论——先给一句话关键发现，再用图表支撑。不同切分维度（司龄/职级/性别）的群体间差距，反映不同的组织结构性问题。

触发：VP 视角报告生成时，在管理者流失风险之后执行本规则。

信号判定逻辑：

| 信号等级 | 条件 | 信号文字 | 颜色 |
|----------|------|----------|------|
| 🔴 预警 | 最差群体健康度 < 65 | "{群体} 健康度仅 {X.X}，需紧急关注" | 红 |
| 🟡 关注 | 最差群体健康度 < 75 或 群体间差距 ≥ 10 | "{群体} 健康度 {X.X}，{与最好群体差距/低于公司均值}" | 黄 |
| 🟢 正常 | 最差群体健康度 ≥ 75 且 群体间差距 < 10 | "各群体健康度均衡（{最低}~{最高}），无明显短板" | 绿 |

分化诊断：
- 司龄分化：新员工（<1年）vs 老员工（>3年）差距大 → 入职融入/职业发展通道问题
- 职级分化：基层 vs 中层差距大 → 管理层信息传递断层
- 性别分化：差距 > 5 分 → 可能存在包容性问题

输出：
- 每个维度一个信号面板（顶部信号框 + 主体横向条形图）
- 条形图按健康度升序排列（最差置顶），灰色轨道 + 彩色条形（四档颜色）+ vs 公司偏离箭头 + 敬业度小字

Rule-017：管理者发展重点 MES 子项短板匹配（VP 视角专用）
原理：经理效能得分（MES）的子项拆解可以定位管理者的具体短板领域，比单一总分更有指导意义。每个二级部门取 MES 子项中最低的 1-2 项，匹配预设建议库，输出可执行的发展建议。

触发：VP 视角报告生成时，在经理效能矩阵之后执行本规则。

计算：
MES = (Q03 + Q04 + Q05 + Q06 + Q07) / 5 × 100 / 6

MES 五个子项：
| 题号 | 子项名称 | 短板阈值 | 预设建议 |
|------|----------|----------|----------|
| Q03 | 发挥优势 | < 70 | 开展优势识别工作坊，帮助管理者了解团队成员优势，优化任务分配 |
| Q04 | 认可表扬 | < 70 | 建立周度团队认可仪式，每项目结束复盘时公开表扬贡献者 |
| Q05 | 主管关心 | < 70 | 推行固定 1v1 沟通节奏（每周≥30分钟），关注员工工作状态与生活平衡 |
| Q06 | 鼓励发展 | < 70 | 为每位员工制定个人发展计划(IDP)，管理者每季度 Review 进展 |
| Q07 | 目标明确 | < 70 | 建立目标对齐机制（OKR/KPI），确保每位成员理解团队目标与个人贡献 |

输出：
- 每个二级部门一张卡片：部门名 + MES 总分 + 短板子项（红色标注）+ 对应改善建议
- 短板子项按得分升序排列，最多展示 2 项

Rule-018：团队成员流失预警（经理人 / 团队负责人视角专用）
原理：个体员工的敬业度 + 绩效 + 管理满意度 + 成长体验形成多维风险画像。经理人 需要监控全团队成员，而非仅管理者（与 Rule-015 VP 视角管理者流失风险互补，后者仅监控 P7+）。

触发：经理人 或三级部门负责人视角报告生成时，在人才九宫格之后执行本规则。

风险分级（敬业度 + 绩效双维核心）：
| 等级 | 条件 | 含义 | 干预策略 |
|------|------|------|----------|
| 🔴 高危 | 敬业度 < 60 或 (敬业度 < 70 + 绩效 C) | 高概率流失 | 7 天内 经理人 + 负责人 1v1 深度访谈，制定保留方案 |
| 🟡 需关注 | 敬业度 < 75 或 (绩效 A/B+ + 敬业度 < 80) | 需密切关注 | 管理者增加 1v1 反馈频次，高绩效人才给予延展性挑战 |
| 🟢 稳定 | 敬业度 ≥ 75 | 常规关注 | 季度成长对话中确认状态 |

多维信号检测（5 类信号）：
| 信号 | 触发条件 |
|------|----------|
| 敬业度极低 | 敬业度 < 60 |
| 敬业度偏低 | 敬业度 < 70 |
| 对管理不满 | 管理支持子维度 < 55 |
| 成长受阻 | 成长体验子维度 < 55 |
| 低绩效 | 绩效等级 = C |
| 高绩效人才敬业不足 | 绩效 A/B+ 但敬业度 < 75 |

输出：
- 顶部三色统计卡（高危人数 / 需关注人数 / 稳定人数）
- 人员卡片列表（仅展示高危和需关注，最多 8 人，稳定人员用统计概括）
- 每张卡片含：匿名编号 / 敬业度 / 管理支持分 / 成长分 / 绩效等级 / 风险等级 / 信号标签集 / 个体行动建议

参考模型：SHRM Talent Retention Framework + Gallup Q12 个体敬业度预测效度。

Rule-019：管理者辅导指南自动匹配（经理人 / 团队负责人视角专用）
原理：针对团队得分最低的 Q12 题目，提供业界验证的对话引导问题和管理者改进行动。不追求面面俱到，聚焦 3 项最薄弱环节给出可执行的辅导动作。

触发：经理人 或三级部门负责人视角报告生成时，在流失预警之后执行本规则。

数据来源：COACH_PROMPTS 预设库（12 题 × 3 字段）：
| 字段 | 说明 | 示例 |
|------|------|------|
| 维度名 | 该题的简短维度标签 | 发挥所长 / 认可与反馈 / 关心个人 |
| 根因分析 | 为什么这一项得分偏低的结构性解释 | "管理者未被要求也不被评估反馈行为" |
| 对话引导问题 | 管理者可在 1v1 中使用的具体话术 | "过去一个月你在工作中发挥了自己最擅长的能力吗？" |

输出：
- 顶部使用说明框（指导管理者 30 天内逐条实践）
- 3 张辅导卡片（按得分升序），每张包含：题号 + 维度名 + 团队得分 + 低于公司差值 + 原始题目文本 + 根因分析 + 对话引导问题 + 30 天行动承诺

Rule-020：团队内部群体分化检测（经理人 / 团队负责人视角专用）
原理：小团队中不同群体（不同司龄/职级/性别）的敬业度差异反映团队内部的结构性问题。分化超过 10 分需要管理者区分对待不同群体。

触发：经理人 或三级部门负责人视角报告生成时，在六大维度对比之后执行本规则。

切分维度与判定：
| 维度 | 分化诊断含义 |
|------|-------------|
| 司龄分化 | 新员工(<1年) vs 老员工(>3年)差距大 → 入职融入/职业发展通道问题 |
| 职级分化 | 基层 vs 中层差距大 → 管理层信息传递断层 |
| 性别分化 | 差距 > 5 分 → 可能存在包容性问题 |

信号阈值：群体间最大敬业度差距 ≥ 10 分 → 输出"⚠ 存在显著群体分化，建议管理者关注低分群体的差异化需求"。

数据要求：子群体成员 ≥ 2 人方纳入分析（n < 2 时该分组不展示）。

输出：
- 团队敬业度均值 + 虚线参考线
- 每个维度一组横向条形图（按敬业度升序，最低置顶）
- 团队均值红色虚线贯穿所有条形
- 偏离值标注（红负绿正）+ 分化信号文字

Rule-021：团队优势与改善机会分析（经理人 / 团队负责人视角专用）
原理：参照 Gallup 方法论，将团队题目得分与公司均值对比，输出"继续保持的优势"和"需要改善的机会"，帮助管理者一目了然。

触发：经理人 或三级部门负责人视角报告生成时，在核心诊断指标之后执行本规则。

判定阈值：
| 区域 | 条件 | 含义 |
|------|------|------|
| 🟢 优势区 | 团队得分 ≥ 75 且 超越公司 > +3 分 | 继续保持并发挥杠杆效应 |
| 🔴 改善机会 | 团队得分 < 65 或 远低于公司 < -5 分 | 优先投入改善资源 |

呈现规则：
- 每区展示最多 TOP 3 项
- 每项附带定制化解读文字（msgs 字典：12 题各一条正面解读 + 一条反面改进建议）
- 无优势时显示友好占位："暂无显著超越公司的优势项。团队在持续改善中，继续加油！"
- 无机会时显示友好占位："暂无显著落后公司的机会项。团队表现均衡！"

输出：双栏布局（绿底=优势区 / 红底=改善机会），每项含题目编号 / 维度名 / 团队得分 / vs 公司差值 / 一句话解读。
Rule-009：局部组织风险
若某三级部门多个指标低于组织平均 0.5 以上 → 判断为局部组织风险。建议优先开展团队诊断。

Rule-010：事业部级风险
若某一级事业部整体低于公司平均 → 判断为事业部级组织问题。重点分析：管理方式、业务压力、资源支持。
